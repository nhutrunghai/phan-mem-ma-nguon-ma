# UX Notes - Public Customer App

Review target: `http://127.0.0.1:8000` public customer-facing flows only. Reviewed home, movies, schedule, movie detail, auth, account demo, and booking seat pages at desktop and mobile widths.

## Prioritized Small Improvements

| Route / Page | Priority | Observation | Small Suggested Improvement |
|---|---:|---|---|
| `/dang-nhap`, `/dang-ky`, `/tai-khoan` when unauthenticated | P1 | The cinema/city selector modal appears immediately and darkens the login/register page before the user can interact with auth fields. This is especially disruptive on mobile where it covers most of the form. | Do not auto-open this modal on auth/account-gated pages, or delay it until the user clicks the cinema selector. |
| `/dat-ve/{movie}` booking seats, mobile | P1 | The seat map is wider than the viewport and is clipped horizontally; users can see partial seats at the left/right edges without a clear scroll affordance. | Add a visible horizontal scroll container cue for the seat map on mobile, such as edge padding plus a subtle "vuot de xem them" hint or fade at the sides. |
| `/dat-ve/{movie}` booking seats | P1 | The "TIEP TUC" button appears active even when no seat is selected, while the sidebar says "Chua chon ghe". | Disable the button until at least one seat is selected and show a short inline helper such as "Vui long chon ghe de tiep tuc." |
| `/phim` and `/lich-chieu` filters | P2 | Search and genre fields have no visible labels, the button label is just "Loc", and there is no count or context after applying filters. | Add compact labels or accessible labels and show a result summary like "0 phim phu hop voi 'zzzzzz'" / "3 lich chieu phu hop". |
| `/phim?q=...` empty result | P2 | The empty state says "Khong tim thay phim phu hop." but provides no recovery action near the message. The "Xoa loc" link remains up in the filter row. | Include a small "Xoa loc" action inside or directly below the empty state message. |
| `/lich-chieu?q=...` empty result | P2 | The empty state says no schedules match, but date tabs remain prominent and can imply results might exist under another date even when the filter excluded everything. | Add the active search/genre terms to the empty message and keep the reset action close to it. |
| `/phim` mobile filters | P2 | On mobile, the search field, genre field, filter button, and reset link stack with uneven spacing; "Xoa loc" reads like body text rather than a related filter control. | Group the reset link with the filter controls and give it button/link styling consistent with the filter action. |
| `/lich-chieu` mobile date selector | P2 | Date tabs become very large and wrap to multiple rows, pushing movie content down and making the selector feel heavier than the schedule content. | Keep date tabs on one horizontal row with overflow scrolling or reduce their mobile type/spacing. |
| `/` home mobile hero/navigation | P2 | The header navigation wraps over two lines and visually overlaps the top of the hero image area, making the first slide feel crowded. | Add a little more vertical separation below the mobile nav or collapse secondary nav items behind the existing menu trigger at this width. |
| `/phim/{movie}` movie detail | P3 | Metadata includes placeholder copy such as "Dao dien: Dang cap nhat" and "Dien vien: Dang cap nhat", which reads unfinished. | Hide metadata rows until real values exist, or use a softer placeholder like "Se cap nhat sau". |
| `/phim/{movie}` schedule slots | P3 | Some slot labels show the date as `29/04/` with a trailing slash, which looks like a formatting mistake. | Remove the trailing slash or render the full intended date consistently. |
| `/tai-khoan` account demo | P3 | The profile page shows "Da dang nhap thanh cong" as a static message after navigation; it can feel stale rather than a recent confirmation. | Use a dismissible toast only immediately after login, then remove it from normal account page refreshes. |
| `/tai-khoan` account demo | P3 | "Chon anh" looks like a wide blue bar but does not clearly communicate upload status or whether the action succeeded. | Add small inline feedback after selection/upload such as filename selected, upload pending, or success/failure text. |
| Global footer links | P3 | Many footer and cinema links point to `#`, so they look interactive but do not lead anywhere or explain unavailable content. | For links not yet connected, either hide them temporarily or add a small unavailable-state message/toast on click. |
